# v0.1.0 Pivot Demo Release Notes

Release tag: `demo-v0.1.0`

Hosted demo target: <https://abdelstark.github.io/verifiable-intelligence/>

## Scope

This release publishes the browser-first proof-market demo from the marketplace pivot. A buyer can select a provider, request a quote, submit a prompt, receive a delivered answer, inspect a `VIEX` proof bundle, and run the browser verifier path over happy and red-path fixtures.

## Static Hosting Path

The hosted demo is published through GitHub Pages by `.github/workflows/static-demo-pages.yml`. The workflow deploys only from tags matching `demo-v*`, so the public page can be traced back to the release tag and commit SHA in `release.json`.

## Provider Runtime

Live provider calls are absent in this release. The hosted static demo uses simulated fixtures and does not call a live GPU endpoint.

The live GPU endpoint cost decision is deferred: the current reference target remains an A100-class host for `llama-3.1-8b-w8a8`, but this release does not reserve, deploy, or bill that endpoint.

## Verification Boundary

- Open-weight only: v1 does not verify closed-weight frontier models.
- Execution integrity only: a passing proof does not certify that the answer is factually correct.
- Lawful-use boundary: no credential collection, account pooling, billing bypass, provider-term evasion, or unauthorized token resale.
- Fixture boundary: simulated fixtures are labeled as fixtures and must not be represented as live CommitLLM provider traffic.

## Release Gate Evidence

- README and demo comprehension copy cover open-weight-only, execution-integrity-only, no unauthorized resale, and fixture-vs-live boundaries.
- Browser tests cover a passing happy path, model substitution, prompt mismatch, answer rewrite, receipt tamper, expired quote, and unsupported closed-weight provider red paths.
